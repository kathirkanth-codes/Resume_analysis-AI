# 🚀 AI Resume Analyzer — Project Overview

## 🎯 Problem Statement

Most resume analyzers today are shallow and ineffective:

* They rely on keyword matching (ATS style)
* They don’t understand role-specific expectations
* They give generic feedback like “improve skills”
* They don’t compare resumes with real strong candidates

👉 Goal:
Build an **AI-powered resume analyzer** that:

* Understands resume content semantically (using embeddings)
* Compares with strong resumes (benchmarking)
* Provides **role-specific insights**
* Gives **actual rewrite suggestions**, not vague advice

---

## 🧠 Core Idea

Instead of checking keywords:
👉 We convert resumes into **vector embeddings** and compare them using **FAISS (vector database)**

This allows:

* Semantic comparison (not just keywords)
* Smarter benchmarking
* Better recommendations

---

## ⚙️ System Architecture

### Flow:

1. User uploads resume (PDF)
2. Backend extracts text
3. Text is cleaned and processed
4. Convert text → embedding (ML model)
5. Store/search in FAISS vector DB
6. Retrieve top similar strong resumes
7. Run analysis engines:

   * Role scoring
   * Benchmark comparison
   * Rewrite suggestions
8. Send results to frontend

---

## 🔄 Workflow (Step-by-Step)

```text
Upload Resume
     ↓
PDF → Text Extraction
     ↓
Text Cleaning
     ↓
Embedding Generation
     ↓
FAISS Search (Top K similar resumes)
     ↓
Role Detection (Data Analyst / Data Scientist / Full Stack)
     ↓
Benchmark Comparison (vs similar resumes)
     ↓
Rewrite Suggestions
     ↓
Final Score + Output
```
